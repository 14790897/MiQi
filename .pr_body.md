## 变更概述

修复 #191：内置 DeepSeek API Key，用户输入激活码即可使用。激活功能嵌入在 Provider 编辑页内，以 radio card 形式提供两种 API 来源选择。

## 背景/问题

新用户需要自行申请 API Key，上手门槛高。企业内测场景需要"激活码 → 自动配置"的零门槛体验，同时不暴露内部 Key。

## 变更内容

**前端 (ProvidersPage.tsx / preload / ipc)**
- DeepSeek EditSheet 新增 API 来源 radio cards：推荐 / 我自己的API Key
- 激活码输入框为密码模式，不显示明文
- 激活后自动验证并设为默认 Provider
- 切换到自己的 Key 时清除 builtin 标志

**后端 (provider_handlers.py / loop.py)**
- 新增 providers.activate 处理器：AES Fernet 解密内置 Key → 写入 provider config
- providers.list 返回 builtin_available / builtin_activated 状态
- 已激活时 api_key_hint 返回"企业共享密钥"，前端永远看不到真实 Key
- providers.update 在用户显式提供 API Key 时清除 builtin 标志

**安全**
- DeepSeek Key 以 Fernet 加密存储在源码中
- 激活码仅在后端代码中出现，前端以圆点遮挡
- 真实 Key 不出现在前端任何地方
